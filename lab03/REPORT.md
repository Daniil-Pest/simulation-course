### Клеточные автоматы. Лесные пожары (GUI)

Константы: `N` (число строк) и `M` (число столбцов). Таблица представлена в виде вектора из векторов — `grid`. Имеется счётчик `tree_count` для числа деревьев (останавливает обновление клеточного автомата).

**Дополнительные правила для поведения системы**:
* пустые ячейки (`EMPTY = 0`) (также конечные состояния) при генерации, не взаимодействующие с другими:

            grid[i][j] = rand() % 2 == 0 ? EMPTY : TREE;


  
            else if (grid[i][j] == FIRE)
            {
                if ((rand() % 100) / 100.0 < 0.1)
                {
                    new_grid[i][j] = ASH;
                }
            }
            else if (grid[i][j] == ASH)
            {
                if ((rand() % 100) / 100.0 < 0.5)
                {
                    new_grid[i][j] = EMPTY;
                    tree_count--;
                }
            }

* направление ветра (`wind_direction`), усиливающее по нему вероятность возгорания в полтора:

                if (wind_direction == 1 && j < M - 1 && grid[i][j+1] == FIRE)
  
                {
  
                    prob *= 1.5;
  
                } else if (wind_direction == 2 && j > 0 && grid[i][j-1] == FIRE)
  
                {
  
                    prob *= 1.5;
  
                } else if (wind_direction == 3 && i > 0 && grid[i-1][j] == FIRE)
  
                {
  
                    prob *= 1.5;
  
                } else if (wind_direction == 4 && i < N - 1 && grid[i+1][j] == FIRE)
  
                {
  
                    prob *= 1.5;
  
                }
  
* влажность (`humidity`) — множитель для вероятности возгорания (от `0` до `1`): `double prob = 0.5 * (1 - humidity)`;
* задержка в остановке горения и рассеивании пепла — по `if ((rand() % 100) / 100.0 < *)`.
